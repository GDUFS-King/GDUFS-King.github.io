先来看下堆块的内存布局：

当注册了一个学生时，先得到一个0x40固定大小的堆块，然后我们可以得到学生用户堆的结构：

chunk{

int size;

string name;

string passwd;

struct *chunk1;//书的堆块

int number; //书的个数

}

再得到书籍堆块：

chunk1{

int size；

string book_name;

…..

}

就是以上两种基础的结构，接着我们可以往书籍的堆块那里填写puts的got表，然后free掉当前学生，再申请2个堆块，依次得到堆块，由于UAF漏洞，所以register得到原book堆块，对应于第五块书籍的位置是我们的book的地址，此时写成了got表，那么调用write函数就可以实现泄露了。
